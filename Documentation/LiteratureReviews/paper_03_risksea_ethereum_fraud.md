# Literature Review: Paper 3

**Paper Title:** RiskSEA: A Scalable Graph Embedding for Detecting On-chain Fraudulent Activities on the Ethereum Blockchain
**Paper Link:** https://arxiv.org/html/2410.02160v1
**Authors:** Ayush Agarwal, Lv Lu, Arjun Maheswaran, Varsha Mahadevan, Bhaskar Krishnamachari
**Affiliation:** Machine Learning Team, Coinbase
**Contact Email:** ayush.agarwal@coinbase.com

**Abstract:**
Like any other useful technology, cryptocurrencies are sometimes used for criminal activities. While transactions are recorded on the blockchain, there exists a need for a more rapid and scalable method to detect addresses associated with fraudulent activities. The paper presents RiskSEA, a scalable risk scoring system capable of effectively handling the dynamic nature of large-scale blockchain transaction graphs. The system consists of: (1) a scalable approach to generating node2vec embeddings for the entire set of addresses to capture graph topology; (2) transaction-based features to capture behavioral patterns; (3) a classifier model that combines node2vec embeddings and behavioral features. Two novel approaches are presented for scaling node2vec to all blockchain addresses: node2vec embedding propagation and dynamic node2vec embedding.

**Introduction:**
The increasing association of cryptocurrencies with various industries has attracted wide attention, but the pseudonymous nature and lack of identity tied to blockchain addresses has enabled criminals to commit cyber crimes. In traditional banking, customers cannot open accounts and transact without direct approval of the financial institution; however, in blockchain, users can transact without approval of regulated entities. As of September 2023, there are approximately 266 million nodes on the Ethereum network. Numerous prior studies have aimed to identify fraudulent transactions on large financial networks, but with few exceptions, prior work has not significantly explored graph embeddings for fraud detection at scale, nor adequately addressed the dynamic nature of blockchain transaction graphs as they evolve over time.

**Motivation:**
Existing fraud detection approaches predominantly model patterns using features derived from transaction timestamps and amounts, then employ these models to uncover previously unnoticed transactions exhibiting distinctive patterns. However, transactional features cannot fully capture the structural patterns or network topology of blockchain addresses. Malicious addresses often disperse their funds within a closely interconnected cluster of supporting addresses. By leveraging node2vec embeddings, these dense clusters can be identified and analyzed for similar transaction patterns—enabling detection and classification of potential malicious addresses based on their network associations. However, standard node2vec has three critical limitations: (1) embeddings cannot be generated for addresses not in training data; (2) generating embeddings for 266M+ nodes is computationally challenging; (3) re-computation is needed for evolving graphs, which is prohibitively time-consuming.

**Methodology:**
The RiskSEA pipeline has three stages. **Data Collection and Preprocessing:** Curated datasets from Coinbase internal data (43,334 data points from KYC'ed customer wallets and blocklisted addresses) and external sources (85,678 data points), with labels: Risk-1/"Bad" (illegal activities) and Risk-0/"Good" (exchanges, miners, government). **Feature Extraction** involves two types: (a) **Behavioral features** — two buckets covering Ethereum transactions and ERC-20 token transactions, capturing statistics on volume, amount, and frequency of incoming/outgoing transactions; (b) **Node2Vec embeddings** — the node2vec algorithm generates low-dimensional representations for graph nodes by constructing random walks and training a semi-supervised model. **Scaling Approaches:** (1) **Embedding Propagation** — train node2vec on a core set of 30M addresses (using NVIDIA V-100 GPU), then for each remaining address X, identify all core addresses X has directly transacted with (one-hop), randomly sample 5 neighbors, and aggregate their embeddings. This generates embeddings for 78% of addresses, with 80–85% coverage for daily-active addresses. (2) **Dynamic Node2Vec** — an incremental training approach inspired by dynnode2vec, where each iteration considers only incremental changes in the transaction graph and updates embeddings only for changed regions. A novel horizontally scalable random walks approach enables efficient incremental training. **Classification** uses a Random Forest classifier combining all three feature sets (behavioral-ETH, behavioral-ERC20, node2vec). Evaluation metrics: Precision, Recall, and PR-AUC on a diverse test dataset.

**Results and Discussion:**
The ablation study reveals clear performance differences across feature combinations. **Behavioral features alone** achieve Precision: 0.803, Recall: 0.650, F1-Score: 0.718. **Propagated Node2Vec embeddings alone** achieve Precision: 0.798, Recall: 0.686, F1-Score: 0.738—graph features alone outperform behavioral features in recall. **Combined (propagated node2vec + behavioral)** achieves Precision: 0.908, Recall: 0.801, F1-Score: 0.851—a substantial boost from either alone. Dynamic node2vec outperforms propagated embeddings because it handles the evolving nature of blockchain (new addresses and transactions added over time) and avoids information loss from propagation aggregation. Node2vec feature importance analysis shows graph-based features have very high importance compared to other features, confirming that network topology captures patterns invisible to transaction-level statistics.

**Conclusion:**
RiskSEA demonstrates that combining graph-based and behavioral features provides the most effective fraud detection for blockchain addresses, achieving F1: 0.851 at scale across all 266M Ethereum addresses. This is the first such comprehensive system deployed in practice. For the Crypto World Bank, this validates the multi-layered risk assessment design where both transaction-level features (amount, frequency, timing) and wallet-level behavioral/network patterns are combined. The scalable dynamic embedding approach is relevant for the project's future deployment where new borrower addresses continuously join the network.
