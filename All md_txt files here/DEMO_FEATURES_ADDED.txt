MOCK DATA LOGIC
================================================================================

The frontend uses simple logic to generate realistic-looking data:

Fraud Score Calculation (Loan page):
- Base score: 0
- +0.3 if amount > 10 MATIC
- +0.2 if purpose < 20 characters
- +0.25 if purpose contains "urgent" or "quick"
- +0.2 if amount > 50 MATIC
- Result capped at 0.95

RL Recommendation (Admin):
- Loan > 10 MATIC → "reject" (78% confidence)
- Loan 3-10 MATIC → "review" (65% confidence)
- Loan < 3 MATIC → "approve" (65% confidence)
- Expected reward: positive for approve, negative for reject

Risk Scores (Risk Dashboard):
- Fraud: 23%
- Anomaly: 35%
- Attack: 12%
- All static values for consistent demo
