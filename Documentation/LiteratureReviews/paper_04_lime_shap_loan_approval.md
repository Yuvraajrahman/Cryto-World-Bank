# Literature Review: Paper 4

**Paper Title:** Comparative Analysis of Explainable AI Frameworks (LIME and SHAP) in Loan Approval Systems
**Paper Link:** https://www.mecs-press.org/ijieeb/ijieeb-v17-n6/v17n6-5.html
**Authors:** Isaac Terngu Adom, Christiana O. Julius, Stephen Akuma, Samera U. Otor
**Affiliation:** Department of Mathematics and Computer Science, Benue State University, Nigeria
**DOI:** https://doi.org/10.5815/ijieeb.2025.06.05

**Abstract:**
Machine learning models that lack transparency can lead to biased conclusions and decisions in automated systems in various domains. To address this issue, explainable AI frameworks such as LIME (Local Interpretable Model-Agnostic Explanations) and SHAP (Shapley Additive Explanations) have evolved by offering interpretable insights into ML model decisions. A thorough comparison of LIME and SHAP applied to a Random Forest model trained on a loan dataset resulted in Accuracy of 85%, Precision of 84%, Recall of 97%, and F1-score of 90%. The study demonstrates that SHAP provides deeper and more reliable feature attributions using Shapley values representing each feature's contribution, that LIME lacks the sophisticated interpretability of SHAP despite offering faster and more generalizable explanations, and quantitatively compares computational efficiency where LIME displays a faster runtime of 0.1486 seconds using 9.14MB of memory compared to SHAP with 0.3784 seconds using 1.2MB.

**Introduction:**
The increasing reliance on machine learning in financial decision-making, particularly in loan approval processes, raises concerns about model opacity and fairness. As Hassija et al. (2024) note, interpreting black-box models is essential for cognitive computing applications. Guidotti et al. (2018) survey methods for explaining black box models, establishing the theoretical foundation for XAI. Erasmo Purificato et al. (2022) study responsible AI techniques specifically in loan approval contexts, arguing that decisions affecting individuals' financial lives must be transparent. Regulatory frameworks including the EU's GDPR mandate the "right to explanation" for automated decisions, making XAI adoption not just beneficial but legally necessary for lending platforms.

**Motivation:**
The widespread adoption of ML for loan approvals creates a fundamental tension between accuracy and interpretability. Complex ensemble models like Random Forest achieve high prediction accuracy but are inherently opaque—neither applicants nor regulators can understand why a specific loan was approved or denied. Diwate et al. (2023) demonstrate that ML can predict loan outcomes effectively but do not address interpretability. Ariza-Garzon et al. (2020) apply explainability to peer-to-peer lending in IEEE Access, finding that model interpretability significantly affects borrower trust. Chen et al. (2023) address interpretable ML for imbalanced credit scoring in the European Journal of Operational Research, showing that class imbalance further complicates explanation quality. This study aims to provide a rigorous quantitative comparison of LIME and SHAP to guide practitioners.

**Methodology:**
A **Random Forest classifier** is trained on a loan approval dataset with features including applicant income, credit history, loan amount, employment status, property area, coapplicant income, and debt-to-income ratio. The dataset is preprocessed with feature encoding and normalization. The trained model achieves Accuracy: 85%, Precision: 84%, Recall: 97%, F1-score: 90%. **LIME** (Ribeiro et al., 2016) generates explanations by creating a local interpretable model around each prediction—it perturbs the input features, observes model output changes, and fits a linear model to the perturbed space. For each loan decision, LIME identifies top contributing features with directionality (positive/negative impact). **SHAP** (Lundberg & Lee, 2017) computes exact Shapley values from cooperative game theory—the average marginal contribution of each feature value across all possible feature coalitions. TreeSHAP is used for the Random Forest model, providing exact (not approximate) feature attributions in polynomial time. Evaluation criteria include: **interpretability depth** (how well each method explains feature interactions), **computational efficiency** (runtime, memory usage), **consistency** (stability of explanations across similar inputs), and **alignment with domain knowledge** (whether identified top features match expert expectations).

**Results and Discussion:**
SHAP consistently identifies credit history, applicant income, and loan amount as the most influential features—aligning with domain expert expectations for loan approval. SHAP feature attributions are additive (they sum to the difference between model prediction and average prediction), providing a mathematically grounded decomposition of each decision. LIME produces reasonable but less stable explanations due to its perturbation-based local approximation—repeated runs on the same instance can yield slightly different feature rankings. Computationally, **LIME runs in 0.1486 seconds using 9.14MB of memory**, while **SHAP takes 0.3784 seconds using only 1.2MB of memory**—LIME is 2.5× faster but uses 7.6× more memory. SHAP's consistency property (a feature that contributes more in one model will never be attributed less importance than in another model) provides stronger theoretical guarantees than LIME's local fidelity. For global explanations, SHAP's bee swarm plots and dependence plots reveal feature interaction patterns that LIME's instance-level explanations cannot capture.

**Conclusion:**
SHAP is recommended for applications requiring deep, consistent, and theoretically grounded interpretability—particularly in regulated financial domains where explanation quality is legally mandated. LIME is suitable for scenarios prioritizing speed and model-agnostic flexibility. This study directly validates the Crypto World Bank's choice of SHAP as the primary explainability method for loan risk assessment, providing bank approvers with Shapley-value-based feature attributions that identify exactly which borrower characteristics (loan amount, repayment history, wallet age, transaction frequency) drove the fraud score, enabling transparent, auditable, and regulatorily-compliant lending decisions.
