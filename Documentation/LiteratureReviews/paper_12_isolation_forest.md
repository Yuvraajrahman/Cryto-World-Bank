# Literature Review: Paper 12

**Paper Title:** Isolation Forest
**Paper Link:** https://ieeexplore.ieee.org/document/4781136
**Authors:** Fei Tony Liu, Kai Ming Ting, Zhi-Hua Zhou
**Affiliation:** Monash University, Australia; Nanjing University, China
**Source:** IEEE International Conference on Data Mining (ICDM), 2008

**Abstract:**
Most existing anomaly detection approaches build profiles of normal instances, then identify instances that do not conform to the normal profile as anomalies. This paper proposes a fundamentally different approach, Isolation Forest, which explicitly isolates anomalies instead of profiling normal data. The method exploits the fact that anomalies are "few and different"—they are susceptible to a mechanism called isolation, requiring fewer random partitions to separate them from the rest of the data. Isolation Forest has linear time complexity O(n), is efficient on high-dimensional data, does not require distance or density computations, and provides an anomaly score with well-defined statistical properties.

**Introduction:**
Anomaly detection methods based on distance (k-NN, LOF), density (DBSCAN, kernel density estimation), or clustering (k-means, Gaussian mixture models) share a common approach: they first construct a profile of what is "normal," then flag instances that deviate. This profile-first approach has fundamental limitations—building accurate normal profiles in high-dimensional spaces is expensive (O(n²) for distance-based methods), profiles are sensitive to parameter choices, and they struggle when normal data is multimodal. The authors propose inverting the approach: instead of modeling normality, directly isolate anomalies. Since anomalies are by definition rare and different, they require fewer random partitions to separate from the rest—this principle forms the basis of Isolation Forest.

**Motivation:**
The specific limitations driving this research are: (1) **computational cost** — LOF requires O(n²) distance computations, making it impractical for datasets with millions of records (such as blockchain transaction logs); (2) **parameter sensitivity** — density-based methods require careful tuning of neighborhood size k, bandwidth h, or minimum cluster size, with poor choices leading to catastrophic performance degradation; (3) **curse of dimensionality** — distance and density become meaningless in high-dimensional spaces (>20 features), but modern fraud detection feature sets commonly have 50–500 features; (4) **labeled data scarcity** — in fraud detection, labeled examples of fraud are extremely rare (<0.1% of transactions), making supervised approaches unreliable. An unsupervised method that works efficiently on high-dimensional data without distance computation is needed.

**Methodology:**
Isolation Forest constructs an **ensemble of isolation trees (iTrees)** through the following procedure: Given a dataset X of n instances with d features, for each tree: (1) randomly sample ψ instances from X (default ψ = 256); (2) recursively partition the sample by randomly selecting a feature q and a random split value p between the feature's minimum and maximum; (3) continue splitting until each instance is isolated (in its own partition) or maximum tree height is reached (log₂ψ). The **anomaly score** for an instance x is computed as: s(x, n) = 2^(−E(h(x))/c(n)), where E(h(x)) is the average path length of x across all trees, and c(n) = 2H(n−1) − 2(n−1)/n is the average path length of unsuccessful searches in a Binary Search Tree (normalizing factor). Scores range from 0 to 1: scores close to 1 indicate anomalies (short average path length), scores close to 0.5 indicate normal instances, and scores below 0.5 indicate very dense clusters. The default ensemble uses **100 trees** with **subsample size 256** and no explicit contamination parameter is needed (unlike one-class SVM). The algorithm's time complexity is **O(t · ψ · log ψ)** for training (t trees) and **O(t · log ψ)** per instance for scoring, making it linear in the number of instances n when ψ is fixed.

**Results and Discussion:**
Evaluation on 12 benchmark datasets (including HTTP/SMTP network intrusion data, satellite imagery, and synthetic anomalies) shows Isolation Forest achieves AUC scores comparable to or better than LOF, Random Forest-based detectors, and one-class SVM, while running **orders of magnitude faster**. On the HTTP dataset (567,497 instances, 3 features, 0.4% anomalies), Isolation Forest achieves AUC: 1.0 with training time of 0.82 seconds, compared to LOF's AUC: 0.98 with training time of 1,469.5 seconds—a 1,792× speedup. On high-dimensional data (Annthyroid: 7,200 instances, 21 features), Isolation Forest maintains AUC: 0.82 while LOF degrades to AUC: 0.71 due to the curse of dimensionality. The subsample size of 256 proves robust across all datasets, eliminating the need for dataset-specific tuning. The only parameter requiring adjustment is the contamination rate (fraction of expected anomalies), which the authors show has minimal impact on performance within a reasonable range (0.01–0.1).

**Conclusion:**
Isolation Forest's combination of linear time complexity, high-dimensional robustness, no distance computation requirement, unsupervised operation, and minimal parameter tuning makes it uniquely suited for real-time anomaly detection in transaction streams. The Crypto World Bank adopts Isolation Forest as its secondary detection model alongside Random Forest: while the supervised Random Forest classifies known fraud patterns, the unsupervised Isolation Forest detects novel anomalous wallet behaviors—abnormal transaction volumes, atypical loan request patterns, or unusual timing distributions—without requiring pre-labeled fraud training data. The algorithm's O(n log n) scoring enables real-time risk assessment during loan request processing.
