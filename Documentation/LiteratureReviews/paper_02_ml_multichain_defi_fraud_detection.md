# Literature Review: Paper 2

**Paper Title:** Leveraging Machine Learning for Multichain DeFi Fraud Detection
**Paper Link:** https://arxiv.org/html/2306.07972
**Authors:** Georgios Palaiokrassas, Sandro Scherrers, Iason Ofeidis, Leandros Tassiulas
**Affiliation:** Department of Electrical Engineering, Yale University, USA; Yale Institute for Network Science, USA; School of Management, Yale University, USA
**Contact Email:** georgios.palaiokrassas@yale.edu

**Abstract:**
Since the inception of permissionless blockchains with Bitcoin in 2008, it became apparent that their most well-suited use case is related to making the financial system available to everyone seamlessly without depending on trusted intermediaries. Smart contracts across chains provide an ecosystem of decentralized finance (DeFi), where users can interact with lending pools, AMM exchanges, stablecoins, and derivatives with a cumulative locked value exceeding 160B USD. While DeFi comes with high rewards, it also carries plenty of risks. The proposed framework introduces an effective method for extracting a set of features from different chains, including Ethereum, and is evaluated over an extensive dataset gathered with transactions of the most widely used DeFi protocols (23 in total, including Aave, Compound, Curve, Lido, and Yearn) based on a novel dataset in collaboration with Covalent. Different ML methods were employed, including XGBoost and a Neural Network, and the introduction of novel DeFi-related features significantly improves evaluation results across Accuracy, Precision, Recall, F1-score and F2-score metrics.

**Introduction:**
Blockchain was originally proposed in 2008 by Nakamoto as the accounting method for Bitcoin cryptocurrency. A milestone for the course of blockchain technology was the development of the Ethereum project, enabling smart contracts that allow the construction of sophisticated on-chain financial systems—DeFi. Several works have combined machine learning and blockchain focusing on both Bitcoin and Ethereum, addressing problems such as Ponzi Scheme detection, address clustering, cryptocurrency price prediction, phishing detection, and illicit accounts detection. Existing works on fraud detection mainly focus on proposing ML methods using labeled datasets and extracting features from transaction history. This paper extends these approaches by considering DeFi protocol interactions and proposing novel features that improve classification results.

**Motivation:**
Previous work on blockchain fraud detection focused on general transactional features without considering the specific behavior patterns of entities interacting with DeFi protocols. Farrugia et al. proposed XGBoost-based detection for illicit accounts using Etherscan labels. Li et al. used 3,559 labeled Ethereum addresses to perform network clustering analysis. Wang et al. proposed DeFiScanner trained on 50,910 DeFi transactions with 37 features. However, none of these works utilized a labeled dataset specifically for DeFi malicious accounts detection. The authors are the first, to their knowledge, to create and utilize a labeled dataset for Ethereum DeFi fraud—identifying all accounts from public malicious address datasets who interacted with major DeFi protocols.

**Methodology:**
The data collection process involves three datasets: (1) **DeFi Transactions Dataset**: 54,000,000+ transactions for 550,000 unique entities interacting with 23 DeFi protocols, collected between blocks 5771740 (May 5, 2019) and 36843000 (March 12, 2023), using an Ethereum full archive node (8-core Intel i7-11700 CPU, 4.8GHz, 32GB RAM, 10TB SSD); (2) **Covalent Decoded DeFi Events**: one of the most advanced industry datasets, indexing 100B+ transactions across 60 blockchains and 200k smart contracts; (3) **Labeled Dataset**: 10,000+ malicious addresses compiled from CryptoScamDB, Etherscan labels, Tether blacklisted addresses, MyEtherWallet blacklist, Farrugia et al.'s dataset of 2,179 accounts, Xblockpro datasets, and Al-E'mari et al.'s labeled transactions. Of these, only 81 malicious addresses also interacted with DeFi protocols, forming the target labeled set. Feature extraction produces **8 transactional features** (total transactions, success rate, standard deviation and range on block height/age, transactions/age ratio, mean/max/std on gas costs) and **414 DeFi-related features** organized as: 24 features for 3 calculations (sum, mean, std) on 8 DeFi event types (add liquidity, remove liquidity, borrow, deposit, liquidation, repay, swap, withdraw); 10 features for 5 calculations on 2 fee types (protocol fees, gas fees); 92 features for 4 calculations on 23 protocols; 44 features for 4 calculations on 11 chains (Arbitrum, Aurora, Avalanche, BSC, Cronos, Ethereum, Fantom, Matic, Moonbeam, Moonriver, Optimism); 144 temporal features aggregating per 1000 blocks (~3.5h); 100 top-token features. SMOTE is applied for class balancing. Five ML classifiers are used: Logistic Regression, Random Forest, SVM (rbf kernel), XGBoost, and ANN (TensorFlow with 5 layers: dense-40-ReLU, 0.3 dropout, dense-10-ReLU, dense-5-ReLU, dense-1-sigmoid, 30 epochs, batch 128, MSE loss, Adam optimizer), with 5-fold cross-validation.

**Results and Discussion:**
All classifiers perform well on "Good" labels with high precision, recall, and F1-Scores. For the "Malicious" label, significant differences emerge. **XGBoost and the Neural Network achieve the highest F1- and F2-Scores**, combining precision and recall effectively: XGBoost reaches Precision: 0.93, Recall: 0.74, F1: 0.85; Neural Network achieves Precision: 0.80, Recall: 0.85, F1: 0.76. Random Forest has high precision but low recall, whereas SVM performs inversely. Logistic Regression performs worst overall. Critically, running the same models using only transactional features (without DeFi features) yields an F1-Score of only 0.08—demonstrating that the 414 DeFi-related features are essential. Feature importance analysis for XGBoost confirms that DeFi-specific features dominate classification decisions, with no single feature standing out but rather many engineered features being utilized—suggesting the classifier leverages information not immediately apparent, which can help detect malicious wallets before they are tagged.

**Conclusion:**
The study establishes that DeFi-specific behavioral features are essential for effective fraud detection in decentralized finance—standard transactional features alone are almost useless (F1: 0.08 vs 0.85). This directly validates the Crypto World Bank's approach of extracting protocol-specific features (loan amounts, borrower history, wallet age, transaction frequency, DeFi event patterns) for its AI_ML_SECURITY_LOG-based fraud detection pipeline. The finding that XGBoost and ensemble methods outperform other approaches supports the project's use of Random Forest as the primary classifier.
